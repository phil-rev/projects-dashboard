# Project Dashboard Knowledge

Team documentation, architecture decisions, and reference material specific to the projects dashboard system.

## Structure

```markdown
# Topic Name

Brief description of what this documents.

## Key Points
- Concept or decision 1
- Concept or decision 2

## Implementation
How to apply this in practice.

## References
Links to related files or external resources.
```

## Naming Convention
- Use kebab-case: `dashboard-architecture.md`, `slack-integration-setup.md`
- Describe the concept or system being documented

## What Goes Here

### System Architecture
How the dashboard works, component interactions, data flow

### Integration Patterns
How the dashboard integrates with Claude Code, GitHub, Slack

### Metadata Standards
What information each project should track in `.project.yml`

### Deployment & Hosting
GitHub Pages setup, GitHub Actions configuration

### Best Practices
How to add projects, maintain metadata, troubleshoot

## Tips
- Keep each doc focused on one topic
- Include real examples when possible
- Link to relevant code files
- Update docs when system changes
- Make docs discoverable for future teams
